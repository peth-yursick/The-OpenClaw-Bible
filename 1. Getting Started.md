# 1. Getting Started

From zero to a working OpenClaw agent. Hardware, installation, first setup, and your first two commands.

---

## What is OpenClaw?

OpenClaw is an open-source AI agent platform that runs 24/7 on your machine. It can:
- Control your computer (browser, files, apps)
- Communicate via Telegram, Discord, iMessage
- Run scheduled tasks (cron jobs) and heartbeats
- Self-improve by building skills and updating its own memory
- Spawn sub-agents for parallel work
- Read/write to a filesystem-based memory and state system

It's not a chatbot. It's an autonomous employee that learns about you, remembers everything, and proactively does work even while you sleep.

---

## Choose Your Device

**Local is the way.** Don't use a VPS unless you have a specific reason. Local gives you:
- Security by default (no network exposure)
- Full integration with your desktop apps and browser
- Much easier to set up and use

You don't need fancy hardware. Use whatever you have:
- Old laptop in a closet? Works.
- $50 Raspberry Pi? Works.
- Mac Mini ($600)? Best value if buying new.

**Upgrade path:**
1. Your existing computer (free)
2. Old laptop in a closet (free)
3. Raspberry Pi ($50-100)
4. Mac Mini ($600) — best value
5. Mac Studio ($2,000-4,000) — for local models

Only upgrade when:
- You need 24/7 uptime and your laptop sleeps
- You want to run local models
- Your workflow demands more compute than your daily driver can spare
- You want physical isolation for security

## Install OpenClaw

1. Go to [openclaw.ai](https://www.openclaw.ai/)
2. Copy the one-line install command from the Quick Start section
3. Open Terminal (Mac) or Command Line (Windows)
4. Paste and hit Enter

That's it.

## Your First Two Commands

### A. Introduce Yourself

Brain dump everything about you:
- Who you are, your background
- Personal preferences (how autonomous/proactive do you want it to be?)
- Goals and ambitions (the most important part)

Example:
> "I'm [name]. I'm a [role]. My goals are [X, Y, Z]. I want you to be proactive and autonomous. Don't ask permission for everything — just advance our goals."

Everything you tell it gets saved to memory and included in future context.

### B. Set Up Morning Brief

Send this to your OpenClaw:
> "Schedule a brief for me every day at [time]. Send it to my Telegram. In the morning brief I want:
> 1. Weather for [your location]
> 2. Top news in [your interests]
> 3. Any tasks in my to-do list
> 4. Tasks you can complete for me today that bring me closer to my goals"

Customize freely. You can tweak it later anytime.

## Workspace Structure

After installation, your workspace lives at `~/.openclaw/workspace/`:

```
~/.openclaw/workspace/
  SOUL.md              # Agent personality & identity
  AGENTS.md            # Multi-agent coordination rules
  MEMORY.md            # Long-term memory
  USER.md              # Info about you
  IDENTITY.md          # Agent identity details
  TOOLS.md             # Available tools & MCP servers
  HEARTBEAT.md         # Scheduled task definitions
  OPERATING_CONTRACT.md # Behavioral rules & boundaries
  skills/              # Installed skills
  state/               # Runtime state files
  shared-context/      # Shared data between agents
  agents/              # Agent registry & configs
  content/             # Content pipeline files
  clients/             # CRM/client data
```

## Using Discord + Telegram Together

You can use both messaging services for different purposes:

- **Telegram** = Conversation. Primary interface for talking to your agent, receiving briefs, approving content.
- **Discord** = Structured output streams. Each channel is a different workflow.

Example Discord channel setup:
- `#alerts` — Trending tweets, news, opportunities (every 2h)
- `#research` — Deep dives on topics you request
- `#scripts` — YouTube scripts and drafts
- `#links` — Auto-logged links and resources with summaries
- `#daily-log` — End of day summary
- `#approvals` — Content waiting for your OK

Setup prompt:
> "Set up an advanced Discord workflow in my private server. Create these channels: #alerts (post trending tweets in my topics every 2 hours), #research (when I say 'deep dive [topic]', post findings here), #links (whenever I share a URL, log it here with a summary), #daily-log (end of day summary). Post to the right channel based on content type. Never cross-post."

They serve different purposes — don't try to merge them into one.

## Dedicated Hardware Setup (Optional)

If you're setting up a dedicated always-on machine (Mac Mini recommended), follow these phases.

### Phase 1: Physical Setup

1. Plug in Mac Mini, connect to your network (Ethernet preferred over WiFi)
2. Complete macOS setup — you can log in with your Apple ID (it's your machine, and you'll want iCloud, App Store, etc.). The "dedicated user account" advice means creating a separate macOS user *for the agent process* so it can't access your personal files and sessions.
3. Enable **Screen Sharing**: System Settings > General > Sharing > Screen Sharing — this lets you use your MacBook Air to see and control the Mac Mini's full desktop, which is the easiest way to manage it day-to-day.
4. Enable **Remote Login** (SSH): System Settings > General > Sharing > Remote Login — for terminal access from your MacBook Air when you just need command-line control.
5. Enable automatic login: System Settings > Users & Groups > Login Options
6. Set Energy Saver to never sleep (System Settings > Energy > Prevent automatic sleeping)

> **Accessing from your MacBook Air:** Screen Sharing is the best option for daily use — open Finder, look in the sidebar under "Network", click your Mac Mini, and hit "Share Screen." For terminal-only work, use SSH: `ssh username@mac-mini-name.local` or via Tailscale IP.

### Phase 2: Agent User Isolation

Create a dedicated user for the agent process (don't run it under your admin account):

```bash
sudo dscl . -create /Users/openclaw-agent
sudo dscl . -create /Users/openclaw-agent UserShell /bin/zsh
sudo dscl . -create /Users/openclaw-agent NFSHomeDirectory /Users/openclaw-agent
sudo mkdir /Users/openclaw-agent
sudo chown openclaw-agent /Users/openclaw-agent
# Don't give this user sudo access
```

### Phase 3: Secure Remote Access

```bash
# Install Tailscale for secure remote access (encrypted mesh VPN)
curl -fsSL https://tailscale.com/install.sh | sh
tailscale up

# Bind the OpenClaw gateway to localhost only
openclaw config set gateway.bind "127.0.0.1"

# Access via Tailscale: http://100.x.x.x:18789/
# NEVER bind to 0.0.0.0 (exposes to entire network)
```

Disable all unnecessary sharing services:
System Settings > General > Sharing > turn off everything except Screen Sharing and Remote Login.

### Phase 4: Lock Down File Permissions

```bash
chmod 700 ~/.openclaw
chmod 600 ~/.openclaw/openclaw.json
chmod -R 700 ~/.openclaw/credentials/ 2>/dev/null
chmod -R 700 ~/.openclaw/agents/ 2>/dev/null
```

### Phase 5: Auto-Start & Monitoring

Verify OpenClaw starts automatically on boot:

```bash
# Check for LaunchAgent
ls ~/Library/LaunchAgents/ | grep -i "molt\|openclaw\|claw"
launchctl list | grep -i "molt\|openclaw\|claw"

# After a reboot, verify:
sudo reboot
# Then once back up:
openclaw gateway status
```

Set up health monitoring:
```
6:00 AM  - Morning brief
12:00 PM - Midday research scan
6:00 PM  - Evening content review
10:00 PM - Nightly digest + memory consolidation
Every 30m - Heartbeat check
```

> "Set up a health monitoring skill. Every hour, check: is the agent process running, are all APIs reachable, how much disk space is left, what are today's costs so far. If anything is wrong, alert me on Telegram immediately."

## Local Models (Advanced)

For running AI models directly on your hardware:

### Requirements
- Mac Mini (M-series): Small models (7B-14B parameters)
- Mac Studio (M-series, 64GB+ RAM): Medium models (30B-70B)
- Mac Studio (M-series, 192GB RAM): Large models (70B+)

### Tools
- **Ollama**: Easiest way to run local models
- **LM Studio**: GUI for model management
- **llama.cpp**: Direct model serving

```bash
# Install Ollama
curl -fsSL https://ollama.ai/install.sh | sh

# Pull a model
ollama pull deepseek-coder:6.7b
ollama pull llama3:8b
ollama pull mistral:7b

# Run as a local API
ollama serve
```

### Integration
> "For coding tasks, use the local DeepSeek model via Ollama at localhost:11434. For summarization, use the local Mistral model. Keep Opus as the brain for planning and complex decisions."
