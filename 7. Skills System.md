# 7. Skills System

Skills are modular capabilities you add to your OpenClaw. Each skill is a folder with a `SKILL.md` file that teaches the agent how to do something specific.

---

## What is a Skill?

A skill = a markdown knowledge pack that gives your agent expertise in a specific domain.

```
.claude/skills/
  tweet-writer/
    SKILL.md          # Instructions, triggers, examples
  morning-brief/
    SKILL.md
  code-reviewer/
    SKILL.md
```

When your agent encounters a task that matches a skill's trigger, it reads that SKILL.md and follows the instructions.

---

## SKILL.md Format

```markdown
# Skill: [Name]

## Description
[One-line: what this skill does]

## Trigger
[When to activate this skill]
- When the user asks about [topic]
- When a scheduled task fires for [task type]
- When [event] happens

## Instructions
[Step-by-step what the agent should do]

1. [Step 1]
2. [Step 2]
3. [Step 3]

## Examples
[Input/output examples so the agent knows what good looks like]

## Rules
- [Constraint 1]
- [Constraint 2]

## Dependencies
- [Required tools: firecrawl, x-research, etc.]
- [Required APIs: Brave, X, etc.]
```

---

## Progressive Disclosure

Don't dump everything into one massive skill. Layer them:

1. **Level 1**: Basic trigger + simple instructions
2. **Level 2**: Add examples and edge cases
3. **Level 3**: Add advanced rules, cost routing, fallback behavior

Start simple, add complexity as you discover edge cases.

## Installing Skills

### From ClawHub / GitHub

```bash
# Install a specific skill
claude install-skill https://github.com/user/repo/tree/main/skills/skill-name

# Example: OpenClaw operations skill
claude install-skill https://github.com/Heyvhuang/ship-faster/tree/main/skills/tool-openclaw
```

### Using npx skills (CLI)

```bash
# Install all skills from a collection
npx skills add username/repo

# Install specific skills
npx skills add username/repo --skill skill-a skill-b

# List available skills
npx skills add username/repo --list
```

### Manual

1. Create a folder in `.claude/skills/`
2. Add a `SKILL.md` file
3. Your agent will discover it automatically

## Building Your Own Skills

When your agent does something poorly or can't do something:

1. Tell it: "Pause. Build a skill to handle this better."
2. It creates a new `SKILL.md` in `.claude/skills/`
3. Future attempts use the skill automatically

**Example flow:**
> You: "Write me a newsletter draft"
> Agent: [writes mediocre draft]
> You: "That's not great. Read all my past newsletters, then build a newsletter skill that captures my style and structure."
> Agent: [reads past work, creates newsletter-writer/SKILL.md with your patterns]
> Next time: [much better output]

This is the self-improvement loop. Use it constantly.

---

## Skill Tips

1. **Clear descriptions matter.** The router uses skill descriptions to decide which skill to activate. Vague descriptions = wrong routing.
2. **Keep skills focused.** One skill per domain. Don't make a "do everything" skill.
3. **Include anti-examples.** Show what bad output looks like so the agent avoids it.
4. **Version your skills.** If a skill works well, don't change it casually. Iterate carefully.
5. **Share what works.** ClawHub is the community marketplace for skills.
