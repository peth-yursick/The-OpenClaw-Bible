# 8. Tools & Plugins

Tools extend what your OpenClaw can do. These are the most useful ones.

---

## Firecrawl

Web scraping and crawling. Extracts clean content from any URL.

**What it does:**
- Scrapes single pages or entire sites
- Returns clean markdown (no HTML junk)
- Handles JavaScript-rendered pages
- Respects rate limits

**Setup:**
1. Get API key from [firecrawl.dev](https://firecrawl.dev/)
2. Install the MCP server or use the API directly

**Use cases:**
- Research: scrape articles, docs, competitor sites
- Content pipeline: pull source material
- Monitoring: check pages for changes

---

## Summarize CLI

Turn any URL, file, or media into a sharp summary.

**Install:**
```bash
npm i -g @steipete/summarize
```

**Usage:**
```bash
# Summarize a URL
summarize "https://example.com/article" --length long

# Summarize a PDF
summarize "/path/to/report.pdf"

# Summarize a YouTube video
summarize "https://youtu.be/..." --youtube auto

# Output as JSON (for automation)
summarize "https://example.com" --json

# Use a specific model
summarize "https://example.com" --model google/gemini-3.1-flash
```

**Features:**
- Supports URLs, PDFs, images, audio/video, YouTube, podcasts
- Extract + summarize pipeline with Firecrawl fallback
- Media pipeline with transcript-first flow and Whisper fallback
- Scriptable output via `--json`, `--extract`, `--metrics`

**Chrome Extension:**
Also available as a Chrome Side Panel extension for one-click summarization. Runs locally via a localhost daemon.

## Camofox Browser

Anti-detection browser for web automation. When regular browser automation gets blocked.

**What it does:**
- Bypasses bot detection (Cloudflare, etc.)
- Manages browser fingerprints
- Rotates user agents and headers
- Handles CAPTCHAs (basic)

**When to use:**
- Scraping sites that block automated browsers
- Social media research that requires login
- Any task where regular browser control gets detected

**Setup:**
Install as an MCP server plugin. Configure in your TOOLS.md.

## X-Research Skill

CLI tool for structured Twitter/X research.

**Capabilities:**
- Search tweets by keyword, user, date range
- Filter by engagement (likes, retweets, replies)
- Find trending topics in specific niches
- Extract thread content
- Analyze engagement patterns

**Search options:**
```bash
# Basic search
x-research search "AI agents"

# Filter by engagement
x-research search "AI agents" --min-likes 100

# User-specific search
x-research search --user @username

# Date range
x-research search "topic" --since 2025-01-01 --until 2025-02-01
```

**Cost:** Uses X API credits. Budget ~$0.01-0.05 per search depending on depth.

## Obsidian CLI

Command-line access to Obsidian vaults. Useful if your agent needs to read/write notes.

**Install:**
```bash
# Install Obsidian CLI (community tool)
npm install -g obsidian-cli
```

**Use cases:**
- Agent reads your notes for context
- Agent writes research findings to your vault
- Daily log entries created by the agent
- Cross-reference between agent memory and your personal notes

**Connecting your vault to the agent workspace:**

```bash
# Symlink your Obsidian vault (or a subfolder) into the agent workspace
ln -s "/path/to/your/obsidian/vault" ~/.openclaw/workspace/obsidian-vault
```

Now the agent can read your notes for context and write research findings, meeting notes, and daily logs directly into your vault.

**Kepano's Official Skills:** The CEO of Obsidian (@kepano) has built multiple skills for Claude Code and Codex that work with both your codebase and your personal vault. Obsidian + AI is becoming a powerful combo — check his GitHub for the latest skills. (Credit: @Hesamation)

## Brave Search API

Web search without Google. Clean, fast, privacy-respecting.

**Setup:**
1. Get API key from [brave.com/search/api](https://brave.com/search/api/)
2. Configure as a muscle for web search tasks

**Use cases:**
- Current events research
- Fact-checking
- Finding documentation
- Competitive analysis

**Cost:** Free tier available. Paid plans for heavy usage.

> **Tip:** Store all tool API keys in `openclaw secrets` rather than `.env` files. See [[3. Security & Hardening#External Secrets Management]].

## Scrapling

Undetectable, adaptive web scraping. Better than Firecrawl for sites with active bot detection.

**What it does:**
- Bypasses bot detection without browser fingerprinting overhead
- Adaptive selectors — doesn't break when sites update structure
- No Cloudflare issues, no selector maintenance
- Pure Python, clean API

**Install:**
```bash
pip install scrapling
playwright install
```

**When to use:**
- Any public site where Firecrawl returns blocked/empty results
- Sites that update their HTML structure frequently (e.g. retail, news)
- Use Firecrawl for complex JS-rendered SPAs and multi-page crawls; Scrapling for simpler scraping with fewer dependencies

(Credit: @hasantoxr)

## OpenClaw Deck

Multi-column terminal session manager for OpenClaw. Built for multi-agent workflows.

**What it does:**

- Sub-agents can spawn within a column

## Spark

Advanced memory layer for OpenClaw. Replaces flat `MEMORY.md` with an intelligence loop.

**What it does:**
- Filters memories by relevance (not everything is worth saving)
- Scores memories by relevance and recency
- Distills: merges related memories, removes noise
- Uses Claude hooks to inject relevant memories into context adaptively
- Fully compatible with Claude and OpenClaw

**Why it matters:** `MEMORY.md` files grow noisy over time. Spark keeps the memory layer clean and ensures what's in context is actually relevant — not just everything ever captured.

(Credit: @meta_alchemist)

## Kilo Gateway

Multi-provider model routing and unified billing. Routes requests to the cheapest capable provider automatically.

**What it does:**
- Unified API for multiple LLM providers (Anthropic, OpenAI, Google, Moonshot, etc.)
- Automatic failover between providers
- Cost tracking and budgeting across all providers in one place
- Latency-based routing — picks the fastest available provider

**When to use:**
- You're using 3+ providers and want unified billing
- You want automatic failover without configuring it per-provider
- You need provider-level cost tracking

**Setup:** Configure as your primary gateway endpoint. Individual provider keys are managed through Kilo's dashboard.

## Pinchtab

High-performance browser control for AI agents. A lightweight alternative to full browser automation frameworks.

**What it does:**
- 12MB Go binary that runs Chrome and exposes a plain HTTP API
- Any agent or script can navigate web pages, read text, click/type, and persist sessions
- Zero config, framework-agnostic, token-efficient
- Designed specifically for agent workflows where you need fast, reliable browser access without Playwright overhead

**When to use:**
- Simple browser tasks (navigate, read, click) where Playwright/agent-browser is overkill
- Scripts that need browser access without a full framework dependency
- Token-sensitive workflows where you want minimal overhead

(Credit: @GithubProjects)

## agent-browser

A better browser control skill for OpenClaw. Replace the built-in browser tool with this.

**Why use it:**
- More reliable than the built-in browser
- Runs on Playwright with full profile support
- Can run headed (visible) or headless
- Works with your personal browser profile (logged in to your accounts)

**Setup (from @wickedguro's SEO machine):**
```bash
# Open with a named profile in headed mode (for login sessions)
agent-browser --profile ~/.myapp-profile open https://semrush.com --headed

# Log in to all required accounts manually once, then automate
```

**For cloud deployment:** Docker Browserless + Patchright + mobile proxy (e.g. BrightData) via CDP. Mobile proxies change IP frequently and are the most undetectable option. Run locally from your Mac Mini instead if possible.

## Tool Management Tips

1. **Only connect tools you actively use.** Each MCP server adds token overhead.
2. **Log tool usage.** Track which tools your agent uses most. Drop unused ones.
3. **Set tool budgets.** Especially for paid APIs. Configure rate limits.
4. **Test tools manually first.** Before letting your agent use a tool, try it yourself to understand its behavior.
5. **Document tools in TOOLS.md.** Your agent reads this file to know what's available.
