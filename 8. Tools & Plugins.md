# 8. Tools & Plugins

Tools extend what your OpenClaw can do. These are the most useful ones.

---

## Firecrawl

Web scraping and crawling. Extracts clean content from any URL.

**What it does:**
- Scrapes single pages or entire sites
- Returns clean markdown (no HTML junk)
- Handles JavaScript-rendered pages
- Respects rate limits

**Setup:**
1. Get API key from [firecrawl.dev](https://firecrawl.dev/)
2. Install the MCP server or use the API directly

**Use cases:**
- Research: scrape articles, docs, competitor sites
- Content pipeline: pull source material
- Monitoring: check pages for changes

---

## Summarize CLI

Turn any URL, file, or media into a sharp summary.

**Install:**
```bash
npm i -g @steipete/summarize
```

**Usage:**
```bash
# Summarize a URL
summarize "https://example.com/article" --length long

# Summarize a PDF
summarize "/path/to/report.pdf"

# Summarize a YouTube video
summarize "https://youtu.be/..." --youtube auto

# Output as JSON (for automation)
summarize "https://example.com" --json

# Use a specific model
summarize "https://example.com" --model google/gemini-3-flash-preview
```

**Features:**
- Supports URLs, PDFs, images, audio/video, YouTube, podcasts
- Extract + summarize pipeline with Firecrawl fallback
- Media pipeline with transcript-first flow and Whisper fallback
- Scriptable output via `--json`, `--extract`, `--metrics`

**Chrome Extension:**
Also available as a Chrome Side Panel extension for one-click summarization. Runs locally via a localhost daemon.

---

## Camofox Browser

Anti-detection browser for web automation. When regular browser automation gets blocked.

**What it does:**
- Bypasses bot detection (Cloudflare, etc.)
- Manages browser fingerprints
- Rotates user agents and headers
- Handles CAPTCHAs (basic)

**When to use:**
- Scraping sites that block automated browsers
- Social media research that requires login
- Any task where regular browser control gets detected

**Setup:**
Install as an MCP server plugin. Configure in your TOOLS.md.

## Superpowers Plugin

Extends Claude Code with additional capabilities.

**Key features:**
- Enhanced file operations
- Better code understanding
- Improved multi-file editing
- Extended context handling

Install as a Claude Code plugin.

---

## X-Research Skill

CLI tool for structured Twitter/X research.

**Capabilities:**
- Search tweets by keyword, user, date range
- Filter by engagement (likes, retweets, replies)
- Find trending topics in specific niches
- Extract thread content
- Analyze engagement patterns

**Search options:**
```bash
# Basic search
x-research search "AI agents"

# Filter by engagement
x-research search "AI agents" --min-likes 100

# User-specific search
x-research search --user @username

# Date range
x-research search "topic" --since 2025-01-01 --until 2025-02-01
```

**Cost:** Uses X API credits. Budget ~$0.01-0.05 per search depending on depth.

## Obsidian CLI

Command-line access to Obsidian vaults. Useful if your agent needs to read/write notes.

**Install:**
```bash
# Install Obsidian CLI (community tool)
npm install -g obsidian-cli
```

**Use cases:**
- Agent reads your notes for context
- Agent writes research findings to your vault
- Daily log entries created by the agent
- Cross-reference between agent memory and your personal notes

**Connecting your vault to the agent workspace:**

```bash
# Symlink your Obsidian vault (or a subfolder) into the agent workspace
ln -s "/path/to/your/obsidian/vault" ~/.openclaw/workspace/obsidian-vault
```

Now the agent can read your notes for context and write research findings, meeting notes, and daily logs directly into your vault.

## Brave Search API

Web search without Google. Clean, fast, privacy-respecting.

**Setup:**
1. Get API key from [brave.com/search/api](https://brave.com/search/api/)
2. Configure as a muscle for web search tasks

**Use cases:**
- Current events research
- Fact-checking
- Finding documentation
- Competitive analysis

**Cost:** Free tier available. Paid plans for heavy usage.

---

## Tool Management Tips

1. **Only connect tools you actively use.** Each MCP server adds token overhead.
2. **Log tool usage.** Track which tools your agent uses most. Drop unused ones.
3. **Set tool budgets.** Especially for paid APIs. Configure rate limits.
4. **Test tools manually first.** Before letting your agent use a tool, try it yourself to understand its behavior.
5. **Document tools in TOOLS.md.** Your agent reads this file to know what's available.
