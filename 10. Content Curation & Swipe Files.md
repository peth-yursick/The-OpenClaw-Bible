# 10. Content Curation & Swipe Files

How to build a content machine: find inspiration, analyze what works, build a swipe file, draft content, and publish — all with your OpenClaw doing the heavy lifting.

---

## The Content Pipeline

```
Research -> Analyze -> Swipe File -> Draft -> Review -> Publish
```

Each stage can be automated to a different degree. The goal: you only touch the "Review" step.

---

## Stage 1: Research (Automated)

Your agent continuously scans sources for content worth paying attention to.

### Sources to Monitor
- **Twitter/X** — Trending tweets in your niche, high-engagement posts, thought leaders
- **Newsletters** — Curated industry content delivered to inbox
- **Reddit** — Subreddit discussions, emerging narratives
- **RSS feeds** — Blogs, publications, competitors
- **YouTube** — Trending videos in your space
- **Hacker News** — Tech and startup discussions

### Setup Prompt

> "Set up a content research pipeline. Every day at [time]:
> 1. Scan Twitter for top 20 tweets in [your niche] from the last 24h (100+ likes)
> 2. Check Reddit [subreddits] for top posts
> 3. Scan my RSS feeds for new articles
> 4. Compile everything into a single research brief
> 5. Save to workspace/content/research/[date].md
> 6. Send me the top 5 most interesting finds on Telegram with one-line summaries"

---

## Stage 2: Analyze (Automated)

Don't just collect content — dissect it. The agent should understand *why* something performed well.

### Tweet Analysis

For every high-performing tweet the agent finds, it should extract:

| Element | What to Capture |
|---------|----------------|
| **Hook** | First line / opening pattern (question, bold claim, number, story) |
| **Structure** | Format used (thread, single tweet, quote tweet, image + caption) |
| **Angle** | Contrarian, educational, personal story, hot take, curated list |
| **Engagement pattern** | What drove replies vs likes vs retweets |
| **Length** | Short punchy vs long-form |
| **CTA** | How they ended (follow, reply prompt, link, nothing) |

### Setup Prompt

> "When you find a tweet with 500+ likes in my niche, analyze it:
> 1. What's the hook pattern? (question, bold claim, number, story, etc.)
> 2. What format is it? (single tweet, thread, image+caption, poll)
> 3. What's the angle? (contrarian, educational, personal, hot take, curated)
> 4. Why did it perform? What emotional trigger or value does it provide?
> 5. Extract a reusable template from the structure
> 6. Save analysis to workspace/content/swipe-file/analyses/[date]-[topic].md
> 7. If this is a new format pattern I haven't seen before, add the template to the swipe file"

### Content & Article Analysis

Same principle for longer content:
- What's the headline pattern?
- How is it structured? (listicle, narrative, how-to, case study)
- What's the hook in the first paragraph?
- What makes it shareable?

## Stage 3: The Swipe File (Curated + Automated)

A swipe file is your library of proven content patterns. It's the single most valuable asset for consistent content creation.

### Structure

```
workspace/content/swipe-file/
  formats.md          # Reusable tweet/post formats & templates
  hooks.md            # Opening lines that work
  angles.md           # Proven content angles for your niche
  examples/           # Full examples of great content (with analysis)
    tweet-[topic].md
    thread-[topic].md
    article-[topic].md
  templates/          # Fill-in-the-blank templates
    hot-take.md
    listicle.md
    story-lesson.md
    contrarian.md
    how-to.md
```

### formats.md Example

```markdown
# Content Formats That Work

## Tweet Formats
### The Bold Claim + Proof
"[Contrarian statement]. Here's why: [evidence]"
Performance: High retweets, polarizing replies

### The Numbered List
"[Number] things I learned about [topic] after [experience]:
1. [Insight]
2. [Insight]
..."
Performance: High saves, steady engagement

### The Before/After
"6 months ago I [old state]. Now I [new state]. Here's what changed:"
Performance: High engagement, aspirational replies

### The Hot Take
"Unpopular opinion: [take]. Most people think [common belief], but [contrarian view because reason]."
Performance: Very high replies, polarizing

## Thread Formats
### The Tutorial Thread
"How to [achieve X] in [timeframe] (step by step):"
-> Each tweet = one step with detail

### The Story Thread
"[Year] ago, I [starting point]. [Dramatic turn]. Here's what happened:"
-> Narrative arc with lessons
```

### hooks.md Example

```markdown
# Hooks That Work

## Question Hooks
- "What if [assumption everyone makes] is completely wrong?"
- "Why do [group] keep making this mistake?"

## Number Hooks
- "[Number] lessons from [experience/timeframe]:"
- "I spent [time/money] on [thing]. Here's what I learned:"

## Contrarian Hooks
- "Stop [common advice]. Here's what actually works:"
- "Everyone says [popular opinion]. They're wrong."

## Story Hooks
- "[Timeframe] ago, I [relatable starting point]..."
- "I almost [dramatic event]. Here's what saved me:"

## Authority Hooks
- "After [impressive credential/experience], here's what I know:"
- "I've [big number] [impressive thing]. Here's what nobody tells you:"
```

### Building the Swipe File

> "Build and maintain a swipe file system. Rules:
> 1. When you analyze a high-performing piece of content, extract the reusable pattern
> 2. Categorize it: format, hook, angle, or full example
> 3. Add it to the appropriate file in workspace/content/swipe-file/
> 4. Include the original source link for reference
> 5. Every Sunday, review the swipe file: remove duplicates, merge similar patterns, rank by how often we've used each
> 6. Track which swipe file patterns I've used and how they performed
> 7. Promote patterns that consistently work; archive ones that don't"

---

## Stage 4: Draft (Automated)

Agent drafts content using your swipe file as a style guide.

### Setup Prompt

> "When drafting content for me:
> 1. First, check the swipe file for relevant formats, hooks, and angles
> 2. Pick a format that fits the topic
> 3. Use a hook pattern from hooks.md
> 4. Write in my voice (reference SOUL.md and past approved content)
> 5. Never sound like AI — no 'leverage', 'utilize', 'game-changing', 'deep dive'
> 6. Keep tweets under 280 chars unless it's a thread
> 7. For threads: each tweet must stand alone AND flow as part of the thread
> 8. End with a natural CTA (not forced 'follow me' — match the swipe file patterns)
> 9. Save draft to workspace/content/drafts/[date]-[topic].md"

---

## Stage 5: Review (You)

This is the only step that requires your attention.

Agent sends drafts to you (Telegram, Discord, or approval queue). You:
- **Approve** — goes to publish queue
- **Edit** — make changes, then approve
- **Reject** — agent learns from rejection (add note why)

> "When I reject a draft, ask me why. Save my feedback. Use it to avoid the same mistake. If I reject 3+ drafts for the same reason, build a rule into the content skill."

---

## Stage 6: Publish (Automated)

Approved content goes to your publishing tool.

- **Typefully** — API integration for scheduled tweets/threads (recommended)
- **Direct X API** — Post directly (more risk, less review buffer)
- **Buffer/Hootsuite** — Multi-platform scheduling

### Typefully Integration

Typefully has an API that lets your agent push approved drafts directly to your scheduling queue.

1. Get your Typefully API key from Typefully settings
2. Give it to your agent
3. Agent pushes approved drafts directly to your Typefully queue

> "When I approve a draft: push it to Typefully via API for scheduling. Log the published content in workspace/content/published/[date].md with the platform and scheduled time. Also log all approved and rejected drafts in Discord #content channel so I have a record."

---

## Performance Tracking Loop

Close the loop: track what performs and feed it back.

> "Every Monday morning, pull engagement stats for everything published last week:
> 1. For each post: impressions, likes, retweets, replies, saves
> 2. Rank by engagement rate
> 3. For the top 3 performers: what swipe file pattern was used? Why did it work?
> 4. For the bottom 3: what went wrong? Should we adjust the pattern?
> 5. Update the swipe file: promote what works, demote what doesn't
> 6. Add any new insights to workspace/content/swipe-file/performance-log.md
> 7. Send me the weekly content performance report on Telegram"

## Information Diet

Not all input is worth processing. Configure your agent to filter aggressively.

### Signal vs Noise Rules

> "When scanning content sources, filter by these rules:
> - **Keep:** Original insights, data-backed claims, novel frameworks, contrarian takes with evidence, personal stories with lessons
> - **Skip:** Repackaged obvious advice, engagement bait without substance, promotional threads, AI-generated filler
> - **Flag for me:** Anything that challenges my current thinking or contradicts my published positions"

### Source Quality Tiers

| Tier | Sources | Action |
|------|---------|--------|
| **Tier 1** (Must-read) | [Your top 5-10 accounts/newsletters] | Always include in brief |
| **Tier 2** (Worth scanning) | [Niche communities, subreddits] | Include if above engagement threshold |
| **Tier 3** (Occasional) | [Broad industry news] | Only if highly relevant |

> "Maintain a source quality list in workspace/content/sources.md. Track which sources consistently produce content I engage with. Promote/demote sources quarterly."

## Periodic Research: New Skills & Use Cases

Stay current on what the OpenClaw community is building.

> "Every week, research what's new in the OpenClaw ecosystem:
> 1. Check ClawHub for new or updated skills relevant to my workflows
> 2. Search Twitter for 'OpenClaw' + 'skill' or 'use case' from the last 7 days
> 3. Check Reddit r/openclaw (or relevant subs) for new workflows people are sharing
> 4. If you find a skill or workflow that would benefit us, add it to workspace/state/skill-candidates.md with: name, what it does, why it's relevant, install command
> 5. Send me a summary on Telegram: 'Found [N] new skills/workflows worth considering: [brief list]'
> 6. I'll tell you which ones to install"

This keeps your setup evolving without you having to manually browse for updates.
