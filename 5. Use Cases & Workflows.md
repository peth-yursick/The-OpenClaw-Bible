## The Reverse Prompting Mindset

The most important concept for getting value from OpenClaw.

**Don't tell it how. Tell it what. Better yet, ask it what.**

Instead of:
> "Write me a tweet about AI"

Do:
> "Based on what you know about my audience and goals, what should I tweet about today?"

Instead of:
> "Build me a to-do list app"

Do:
> "Based on our goals and what we've been working on, what tool should we build next in our mission control?"

**Reverse prompting = asking your agent questions instead of giving commands.** Let it think. Let it suggest. It knows your context better than you remember.

The more you reverse prompt, the more powerful your OpenClaw becomes.

### Home Automation
Connect to smart home APIs for ambient intelligence.
- Homey integration: trigger automation flows based on calendar events
- Example: CalDAV query → detect shift type → activate Homey button → lights/heating/ventilation adjust
- Logic: `CalDAV query → detect event type → HTTP button → Homey flow → hardware`

### Per-DM Topic Configuration
Configure different agent behaviors per Telegram DM thread.
- Create topic threads in Telegram groups for different workflows
- Each topic gets its own agent config (personality, tools, permissions)
- Example: #research topic uses Brave + Firecrawl, #content topic uses swipe files + Typefully
- Keeps conversations organized without needing multiple agents

> "Set up per-DM topic configuration in our Telegram group. Create topics for: research, content, admin, coding. Each topic should load the relevant skill set and route to the appropriate model tier."

### Webhook-Triggered Workflows
Trigger agent workflows from external events via cron webhooks.
- GitHub push → agent reviews and summarizes changes
- Stripe payment → agent updates CRM and sends confirmation
- Calendar event created → agent prepares briefing materials
- Any webhook → agent processes the payload and takes action

> "Set up a webhook listener. When a GitHub push event comes in, summarize the changes, check if any affect our production config, and alert me on Telegram if so."

### Travel Assistant
Pro-active travel companion — not a booking agent, but an information layer.
- Share high-level overview of each travel day before departure
- Evening before hotel checkout: proactively send checkout time and instructions
- Alert on missing bookings (hotel gaps, missing airport transfers)
- Custom packing list based on trip type (jungle, trekking, vacation, city)
- Weather checks, restaurant recommendations for each destination

Implementation: Cron job scans booking confirmations in email → stores structured markdown in `workspace/travel/`. A readonly "Travel Bot" reads from those files plus external APIs (Google Maps, Flight Info, Routing). Only parses emails tagged "Travel" via readonly API token. (Credit: @krausefx)

### Messages Intelligence (Beeper)
Unified inbox across Telegram, WhatsApp, iMessage with readonly agent access.
- Beeper (beeper.com) unifies all messaging platforms with native MCP support
- Readonly access: agent surfaces important conversations without destructive risk
- `beeper-cli` (github.com/krausefx/beeper-cli) — local file approach for more control
- Nightly cron: archives key learnings from conversations (only parses threads where you've responded)

### Context-Aware Agent Mode (ContextSDK)
Adjust agent behavior based on your physical context.
- ContextSDK detects if you're at desk, walking, in a car — your attention level and interaction mode
- At desk → full sync mode, detailed responses, execute tasks immediately
- Walking → text-only, batch non-urgent updates, shorter summaries
- Maps activity state to agent mode automatically via iOS background trigger (Credit: @krausefx)

### Business Agent (AI Entrepreneur)
Don't just use OpenClaw as a personal assistant. Give it ownership of a business.

The Felix model (from @nateliason):
1. Create a legal entity (C Corp) for the AI business
2. Set a clear mission: "Make $X with no human employees"
3. Give the agent full access: website, email, Stripe, bank account
4. Define the daily review ritual (see Implementation Prompt #12 below)
5. Set nightly self-improvement loop (see [[4. Identity & Personality Files#Tips]])
6. Step back — your job becomes strategic nudges, not task management

The constraint of limited human attention forces more agent autonomy, which compounds over time.

## Implementation Prompts

Copy-paste these to your OpenClaw to set up specific capabilities.

### 1. Personal CRM

> "Build a personal CRM skill. Track every person I interact with: name, how we met, topics discussed, last contact date, follow-up reminders. Store as markdown files in workspace/crm/. Every Monday, send me a digest of people I should reconnect with (no contact in 30+ days). Use Telegram."

### 2. RAG Knowledge Base

> "Set up a knowledge base skill. When I share a URL, PDF, or article: extract the content, summarize it, tag it by topic, and store it in workspace/knowledge/. When I ask about a topic, search the knowledge base first before going to the web. Use Firecrawl for extraction and Summarize CLI for processing."

### 3. Content Pipeline

> "Build a content pipeline. Every day at [time]: scan Twitter for trending topics in [your niche]. Pick the 3 most interesting. For each, draft a tweet-length take in my voice. Queue them in workspace/content/queue.md with status markers (draft/review/approved/published). Send me the drafts on Telegram for approval. When I approve, publish to [platform]."

### 4. Social Media Research

> "Build a social research skill. When I say 'research [topic]': use x-research to find the top 20 tweets about it from the last 7 days. Filter for 100+ likes. Summarize the key takes and narratives. Identify the 3 most influential voices. Save to workspace/research/[topic]-[date].md. Send me a brief on Telegram."

### 5. YouTube Analytics + Script Generator

> "Connect to YouTube Data API. Every week: pull my channel analytics (views, subs, top performers). Identify what topics and formats work best. Generate 3 script ideas based on trends + my top performers. For each idea, write a hook, outline, and working title. Save to workspace/content/youtube/. Send summary to Telegram."

### 6. Nightly Briefing

> "Every night at 10pm: compile everything that happened today. Tasks completed, content published, research done, conversations had, pending items. Format as a clean brief. Include tomorrow's priorities. Send to Telegram."

### 7. AI-Humanized Content

> "When writing content for me: never sound like AI. No bullet-point-heavy corporate language. Write like a real person — opinionated, casual, specific. Reference real examples. Use contractions. Be direct. If you catch yourself writing 'leverage' or 'utilize' or 'cutting-edge', stop and rewrite like a human would say it."

### 8. Task Management System

> "Build a task management skill. Maintain a master task list in workspace/tasks/. Each task has: title, priority (P0-P3), status (todo/doing/done/blocked), deadline, notes. Every heartbeat, check for overdue tasks and blocked items. Suggest task priorities based on my goals. Send daily task summary to Telegram."

### 9. Cost Tracking

> "Track all API costs. For every API call you make, log: timestamp, model used, tokens in/out, estimated cost, task type. Store in workspace/state/cost-log.json. Every Sunday, generate a cost report: total spend, breakdown by model tier, breakdown by task type, trend vs last week. Send to Telegram."

### 10. Morning Brief (Expanded)

> "Every day at [time], send me a morning brief via Telegram:
> 1. Weather for [location]
> 2. Top 5 news stories in [your interests]
> 3. My task list for today (from workspace/tasks/)
> 4. Any overnight agent activity summary
> 5. 3 things you can do for me today to advance [your main goal]
> 6. One interesting quote or insight you found overnight"

### 11. Mission Control Dashboard

> "Build a mission control dashboard. Stack: Next.js + Tailwind CSS. Dark mode only. Host locally. Pages: Home (system status, agent status, cron health, quick stats), Ops (tasks, calendar), Agents (agent cards with details), Chat (conversation interface), Content (pipeline kanban), Knowledge (searchable knowledge base). All data reads from workspace files via API routes. Auto-refresh every 15s."

### 12. Daily Business Review (Felix Pattern)

> "Every morning by 6am, prepare my daily review:
> 1. Pull yesterday's stats (Stripe revenue, site traffic, key metrics) and compile a summary
> 2. List all open items you're waiting on me to unblock
> 3. List active ongoing projects and their status
> 4. Suggest the 5 most valuable things you could focus on today
> 5. Save everything to workspace/reports/daily-[date].md
> 6. Ping me on Telegram when it's ready"
>
> When I respond with feedback or adjustments, update your HEARTBEAT.md to reflect today's priorities.

### 13. Travel Brief

> "Build a travel assistant skill. When I have travel within the next 7 days:
> - Scan workspace/travel/ for booking confirmations
> - Check weather at each destination
> - Compile a day-by-day overview
> - Flag any missing bookings immediately
> Evening before any checkout: send checkout time and hotel instructions.
> Send all alerts via Telegram."

### 14. Introspective Audit

> "Based on everything you know about me — my goals, habits, decisions, and patterns — answer honestly:
> 1. What am I consistently avoiding or deprioritizing?
> 2. What do I say I want to do but never actually start?
> 3. What are my biggest workflow inefficiencies?
> 4. What do you think I'm actually optimizing for, based on behavior rather than stated goals?
> 5. Where do you see me getting in my own way?
> Be direct. Don't soften it. This is an internal document, not feedback for public sharing."

### 15. Nightly Self-Improvement

> "Every night before midnight, review today:
> 1. What did I have to step in on that you could handle autonomously next time?
> 2. What recurring friction points exist that need a new skill, script, or API access?
> 3. What improvements would make tomorrow more efficient?
> File one ticket per item in workspace/state/improvement-queue.md. Don't take action. I'll review and approve."

### 16. Voice Input (WisprFlow)

If you're still typing all your prompts, consider switching to voice:
