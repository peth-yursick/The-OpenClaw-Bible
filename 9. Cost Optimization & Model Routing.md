# 9. Cost Optimization & Model Routing

Running AI agents 24/7 gets expensive fast. These strategies cut costs 5-10x without losing quality.

---

## The Model Hierarchy

Not every task needs your smartest (most expensive) model. Route tasks to the cheapest model that can handle them.

### Three Tiers

| Tier | Models | Cost | Use For |
|------|--------|------|---------|
| **Tier 1** (Cheap) | DeepSeek, Haiku, Gemini Flash, MiniMax, Kimi K2.5 | $0.10-0.50/M tokens | Summarization, classification, simple Q&A, formatting, data extraction |
| **Tier 2** (Mid) | Sonnet 4.5, GPT-4o-mini, Gemini 3.1 | $1-3/M tokens | Coding, content writing, analysis, moderate reasoning |
| **Tier 3** (Premium) | Opus 4.6, GPT-4o, Claude 4.5 | $10-15/M tokens | Complex planning, nuanced decisions, creative work, multi-step reasoning |

### Routing Rules

Configure your agent to auto-route:

```markdown
# Model Routing

## Default: Tier 2 (Sonnet)
Use for most tasks unless specified below.

## Route to Tier 1 when:
- Summarizing content
- Reformatting text
- Simple data extraction
- Classification tasks
- Translating between formats

## Route to Tier 3 when:
- Strategic planning or decision-making
- Complex multi-step reasoning
- Creative writing that needs voice/personality
- Debugging subtle or complex issues
- Evaluating tradeoffs
```

### Budget Example

Without routing: Everything on Opus = ~$50-200/month
With routing: 70% Tier 1, 25% Tier 2, 5% Tier 3 = ~$10-30/month

## ClawRouter

OpenClaw's built-in cost-optimizing model router.

**What it does:**
- Analyzes each task's complexity
- Routes to the cheapest capable model
- Falls back to higher tiers on failure
- Tracks costs per task type

**Failover chain:** Tier 1 -> Tier 2 -> Tier 3
If a cheap model fails or gives bad output, automatically retry with the next tier.

### Kilo Gateway for Multi-Provider Routing

If you're using 3+ providers, consider Kilo Gateway as your routing layer. It unifies billing across all providers and adds automatic failover — if Anthropic is down, it routes to OpenAI seamlessly.

The cost benefit: Kilo tracks per-provider pricing and can route to the cheapest provider for each model tier. Combined with ClawRouter's task-complexity routing, you get two-level optimization: right model tier AND cheapest provider for that tier.

## Provider Spending Limits

Set spending limits on each provider to avoid surprises.

### Moonshot (Kimi K2.5)
- Currently free through nvidia (Feb 2026)
- Dashboard: https://build.nvidia.com/moonshotai/kimi-k2.5
- No auto-charge: when credits run out, API calls fail (safe by default)

### Anthropic
- Set daily + monthly limits: Settings > Plans & Billing > Spending Limits
- Recommended: $5/day, $50/month to start
- Enable email alerts at 50% and 80% of limit

### OpenAI
- Set monthly budget: Settings > Organization > Billing > Usage Limits
- Hard limit = API calls fail when reached
- Soft limit = email notification only

### Monitoring

```bash
# Check current usage across all providers
openclaw status --usage
```

## Token Optimization Tips

### Claude 4.6 Adaptive Thinking Costs

Claude 4.6's adaptive thinking adjusts reasoning depth per task. This is great for quality but has cost implications:

- **Simple tasks:** Minimal overhead — adaptive thinking keeps it lean
- **Complex tasks:** May use 2-3x more tokens than a non-adaptive response as the model reasons more deeply
- **Net effect:** Usually cost-neutral because you get better first-attempt results (fewer retries)

**Tip:** If you notice a cost spike, check if complex planning tasks are triggering deep reasoning. You can route those to Sonnet for lighter reasoning when Opus-level depth isn't needed.

### 1. Session Hygiene
Start fresh sessions for new topics. Long sessions accumulate context that costs tokens on every message.

### 2. Lean Context Loading
Don't load your entire workspace into context. Only reference files needed for the current task.

### 3. MCP Server Costs
Each connected MCP server adds its tool descriptions to every request. That's hundreds of tokens per message. Only connect what you need.

```markdown
# Bad: 12 MCP servers connected, using 3
# Good: 3 MCP servers connected, using 3
```

### 4. Skill Description Quality
Clear, concise skill descriptions = faster routing = fewer wasted tokens exploring wrong skills.

### 5. Batch Work
Instead of 10 separate small requests, batch them into one larger request. The per-request overhead (system prompt, tool descriptions) is paid once instead of ten times.

### 6. Compress Memory
Periodically tell your agent: "Consolidate your memory files. Remove duplicates. Keep only what's actively useful." Smaller memory = fewer tokens per session.

### 7. Haiku for Heartbeats
Heartbeats run frequently (every 30 minutes by default). At scale, those checks add up fast. Route them to the cheapest capable model.

Real-world routing from a production OpenClaw setup (via @nateliason's Felix):
- **Opus 4.6 + Codex** — substantive work: planning, coding, content creation
- **Haiku** — heartbeats, stats checks, routine monitoring

Add to your `HEARTBEAT.md`: `"All heartbeat checks use Haiku unless a task is complex enough to escalate."`

The rule of thumb: if the task is "check if anything needs doing" rather than "do something complex," use Haiku.

### 7. Local Models for Muscles
Run cheap tasks locally for $0:
- Summarization on local Llama/Mistral
- Code formatting on local DeepSeek
- Classification on local small models

Requires decent hardware (see [[1. Getting Started#Local Models (Advanced)]]).

### 8. Nested Sub-Agent Costs

Nested sub-agents multiply your token usage. Each level of nesting is a separate model call:

- Coordinator → 3 sub-agents → each spawns 2 workers = **9 model calls** for one task
- If all are on Opus, that's 9x Opus pricing

**Mitigation:**
- Route sub-agents to cheaper tiers (coordinator on Opus, workers on Haiku)
- Set depth limits: `openclaw config set agents.defaults.subagent.maxDepth 2`
- Only use nesting when the task genuinely needs multi-level decomposition

## Cost Tracking

Monitor what you're spending:

> "Track all API costs. Log each request with: model used, tokens in/out, estimated cost, task type. Give me a weekly cost report."

This lets you identify:
- Which tasks are unexpectedly expensive
- Where routing could be improved
- Whether specific workflows need optimization
